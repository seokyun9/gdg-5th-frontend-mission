## 2번째 미션 : 동기/비동기 처리 학습하기

### 왜 이 미션이 중요할까요?

동기 처리는 요청이 끝날 때까지 다음 작업을 멈추고 기다리는 직렬적인 작업 방식이고, 비동기 처리는 요청의 완료를 기다리지 않고 다음 코드를 실행하는 병렬적인 작업 방식이에요.

웹에서 외부 서버와 통신할 때는 필연적으로 네트워크 지연(딜레이)이 발생해요. 비동기 처리를 사용하지 않으면 통신이 끝날 때까지 웹 페이지가 멈추거나, 요청한 데이터가 도착하기 전에 다음 코드가 실행되어 원치 않은 값(에러)에 접근하는 문제가 생길 수 있어요. 이런 상황을 우아하게 해결하는 방법을 함께 공부해요.

## 4번째 미션 : sampleAPI를 이용하여 api 연결해보기

### 왜 이 미션이 중요할까요?

실제 백엔드와의 데이터 연결은 프론트엔드 개발의 가장 필수적인 부분이에요. 다음 주차의 연결 세션에 대비하여, 외부의 공개된 API를 이용해 서버와 데이터를 주고받는 과정을 직접 실습해 봐요.

이번 미션은 **두 가지 난이도의 사이트**를 제공해 드려요. 첫 번째 방법을 먼저 시도해 보는 것을 추천하고, 어렵다면 두 번째 방법을 선택해 API 연결을 완료해 봐요.

### 미션 내용 - 첫 번쨰 방법

1. 사이트 방문하기 : [Reqres](https://reqres.in/signup) 사이트에 접속하여 API Docs 에 들어가 Swagger UI가 있는 API 명세 페이지로 이동해요.

2. 데이터 구조 및 타입 확인

    두 번째 GET 요청은 사용자 목록을 반환하는 API예요. 응답 섹션을 펼쳐 API가 반환하는 데이터 구조를 확인해요. 반환되는 데이터의 구조는 다음과 같습니다.

    ```json
    {
        "page": 0,
        "per_page": 0,
        "total": 0,
        "total_pages": 0,
        "data": [
            {
            "id": 0,
            "email": "string",
            "first_name": "string",
            "last_name": "string",
            "avatar": "string"
            }
        ]
    }
    ```


3. 데이터 가져오기 : 3번째 미션에서 학습한 async/await로 이루어진 함수에 fetch를 이용하여 데이터를 불러와 주세요. 

4. 개별 속성 출력 : 불러온 JSON에서 `data` 속성 안에 있는 사용자 목록 배열만 추출해야 해요. 불러온 데이터를 순회하면서 각 객체의 `email`, `first_name`, `last_name` 등 속성 하나하나씩을 화면에 분리하여 출력하는 코드를 구현해 봐요.

5. 스타일링 : 출력된 목록이 회원 목록 카드 형태처럼 보이게 css를 활용하여 꾸며보면 더욱 좋아요!! 이건 필수는 아니지만 그냥 속성을 나열만 하지는 말아주세요

### 미션 내용 - 두 번쨰 방법

1. 사이트 방문하기 : [sampleAPI](https://sampleapis.com/) 사이트의 coffee 카테고리 데이터를 가져와 출력해보려고 해요. 

    해당 페이지에 들어가 API 주소와 출력 결과를 확인해요.

2. 데이터 구조 및 타입 확인

    API 호출 결과로 다음과 같은 객체 배열을 받게 됩니다. 각 객체는 음료 하나의 정로를 담고 있어요.

    ```javascript
    {
        "title" : string;
        "description": string;
        "ingredients": [문자열 배열];
        "image": string;
        "id": number
    }
    ```


3. 데이터 가져오기 : 3번째 미션에서 학습한 async/await로 이루어진 함수에 fetch를 이용하여 데이터를 불러와 주세요. (사이트에 있는 코드를 참고해 구현하면 훨씬 쉬울거예요..!!)

4. 개별 속성 출력 : 단순히 JSON 파일 전체를 출력하는 것이 아니라, 불러온 데이터를 순회하면서 각 객체의 `title`, `description`, `ingredients` 등 속성 하나하나씩을 화면에 분리하여 출력하는 코드를 구현해 봐요.

5. 스타일링 : 출력된 목록이 메뉴판처럼 보이도록 css를 활용하여 꾸며보면 더욱 좋아요!! 이건 필수는 아니지만 그냥 속성을 나열만 하지는 말아주세요

## 추가 미션 : 쇼핑몰 웹 기능 구현하기

### 왜 이 미션이 중요할까요?

이번 미션은 쇼핑몰 웹 구현의 마지막 페이지인 관리자 페이지를 구현하는 과정이에요. 지난 주차에 배운 Hooks (특히 useState)의 개념과 사용법을 익숙하게 만들기 위한 중요한 실습이므로, 끝까지 포기하지 말고 구현해 봐요!


### 미션 내용
**관리자 기능 구현하기**

1. 관리자 페이지 Navbar 구현하기

    - 소비자 페이지와 마찬가지로 관리자 페이지 상단에도 Navbar가 필요해요.

    - 새로운 관리자 전용 Navbar를 만들거나, 기존 Navbar를 수정하여 사용해 보세요. 기존 Navbar 수정은 개발에서 흔히 쓰이는 방법이며, **useState**를 이용해 현재 페이지가 소비자용인지 관리자용인지를 파악하고 **조건부 렌더링**을 사용하면 쉽게 구현할 수 있어요.


2. 상품 등록 구현하기

    - 상품명, 수량, 가격, 카테고리를 입력하고 등록 버튼을 누르면 콘솔창에 해당 정보와 함께 "등록되었습니다." 가 출력되게 만들어요.

        (예시 : "청바지 5 20000 의류 가 등록되었습니다.")

3. 재고 추가 구현하기

    - 상품명과 수량을 입력하고 추가 버튼을 누르면 콘솔창에 해당 정보와 함께 "추가되었습니다."가 출력되게 만들어요.

        (예시 : "원피스 5 가 추가되었습니다.")

4. 상품 삭제 구현하기

    - 상품명을 입력하고 삭제 버튼을 누르면 콘솔창에 해당 정보가 함께 "삭제되었습니다" 가 출력되게 만들어요.

        (예시 : "청바지 가 삭제되었습니다.")
