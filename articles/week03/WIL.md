# week03 WIL
3주차   

### 미션 1
미션 1 내용은 데이터통신 과목 내용과도 연결점이 있고 지난 주에 학습했던 HTTP의 연장선이었던 것 같다.   
다만 토큰에 대해서는 아직 잘 이해하지 못한 것 같아 시간을 내서 다시 학습해보려한다.

### 미션 2
미션 2에 시간을 제일 많이 할애한 것 같은데 블로그로 작성하느라 그런건지 아니면 내용자체가 많고 복잡해서 그런지는 잘 모르겠다.   
콜백 함수는 얼마 전에 다른 내용을 보다가 궁금해서 찾아봤던 것 같은데 그때 너무 어려워서 살짝 미뤄두었던 기억이 있다.
  
그때 좀 제대로 공부해둘걸 이라는 생각을 하며 공부했다.    
다시봐도 어려운 건 매한가지였다.


동기-비동기 개념과 블로킹-논블로킹 부분도 정말 어려웠다.    
동기가 블로킹이고 비동기가 논블로킹인 거 아닌가? 라는 생각이 끝없이 머리를 지배했는데 개념이 지향하는 관점 측면에서 바라보고 여러 예시들을 찾아보니 조금은 이해가 가는 것 같다.    
물론 아직 완전히 이해한 것은 아니다.    

promise,callback,async,await 부분은 아직도 잘 모르겠고 너무 어렵지만 꼭 알아야하는 내용이니 다시 천천히 공부해야겠다.

미션 2는 시간도 많이 투자하고 글도 정말 여러 글을 찾아 읽어보았는데 아직 내 언어로 잘 정리가 안되는 것 같아서 나중에 블로그 글을 다시 리팩토링하듯... 뜯어고쳐보고 싶다.   

### 미션 3
미션 3는 살짝 뒤로 갈 수록 힘이 빠지기도 하고 미션 4와 서브미션 구현을 빨리 해치우고 싶어서 그냥 가볍게 공부했던 것 같다.   
그렇지만 중요한 내용이기도 하고 생각보다 잘 이해가 안되어서 특히 fetch()함수 부분은 제대로 다시 정리해보아야 할 것 같다.    

### 미션 4
미션 4에 대해서는 일단 api key 부분에서 애를 많이 먹었다.    
하루 날 잡고 했으면 오히려 빨리 했을 것 같은데 일단 url 받아두고 다른 미션들 공부한 다음에 미션 4를 다시 시작한게 패착이었다.   
사이트 앞쪽에 사이트 이용법과 키를 주는 부분을 못봐서 401에러가 떴을 때 어디서 키를 가져와야하는지 찾느라 꽤 시간을 썼다.    
괜히 프록시 설정도 보고 관리자모드 네트워크 부분도 보고... 그래도 굳이 자세히 보지 않았던 request 메시지나 response 메시지를 직접 열어보는 경험을 한 것 같다.  

API 연동은 아직 너무 어렵기도 멀게도 느껴진다.   
말로만 듣던 모호한 개념 비동기 처리/ async / await도 아직 어색한데 fetch 함수부터 콜백함수 promise 등등 너무 많은 개념들이 한 번에 들어와 아직은 잘 정리가 안된 것 같다.    

또한 개념만 대충 알지 직접 코드를 짜보라고 하면 꽤나 버벅이게 된다.   
이리저리 찾아보며 적용해보느라 잘 짜고 있는 건지도 모르겠고 아직 코드의 의미를 완전히 이해하지 못하는 것 같다.   
코드를 눈으로 봤을 때 무슨 의미인지, 흐름과 의도를 잘 파악하지 못하니 답답하기도 하고 공부해야할 부분이 많다는 것을 느꼈다.  


지난 미션에 비해 이번 주 미션은 개인적으로 노력과 시간 투자가 부족했던 것 같아서 아쉬움이 남는다.   
미션코스가 벌써 반 이상 지나갔는데 솔직히 얼마나 성장했는지 아직 잘 모르겠다.
API 연결 세션과 남은 코스도 무사히 마무리할 수 있으면 좋겠다~
